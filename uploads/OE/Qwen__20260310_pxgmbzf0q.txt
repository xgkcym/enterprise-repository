┌─────────────────────────────────────────────────────────────┐
│                      用户请求入口                            │
│  POST /chat { "user_id": "u123", "query": "xxx", "session_id": "s456" } │
└────────────────┬────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  节点 0: 请求预处理 (Preprocessor)                           │
│  ├─ 输入: raw_request                                        │
│  ├─ 处理:                                                    │
│  │  • 参数校验 (pydantic)                                    │
│  │  • 用户鉴权 (JWT + Redis 黑名单)                          │
│  │  • 敏感词过滤 (DFA 算法)                                  │
│  │  • 请求限流 (Redis + 滑动窗口)                            │
│  ├─ 输出: validated_request {user_id, query, session_id, timestamp} │
│  └─ 异常: 400/401/429 直接返回，不进入后续流程                │
└────────────────┬────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  节点 1: 记忆加载器 (Memory Loader)                          │
│  ├─ 输入: validated_request                                  │
│  ├─ 处理:                                                    │
│  │  【短期记忆】                                              │
│  │  • key: f"session:{session_id}:messages"                 │
│  │  • 操作: Redis LRANGE 0 -1 → 取最近 50 条                 │
│  │  • 截断: 按 token 计数 (tiktoken)，超 4K 则摘要压缩        │
│  │                                                          │
│  │  【长期记忆 - 用户画像】                                   │
│  │  • key: f"user:{user_id}:profile"                        │
│  │  • 操作: PostgreSQL SELECT preferences, facts WHERE user_id │
│  │                                                          │
│  │  【长期记忆 - 向量检索】                                   │
│  │  • 操作: Pinecone query(                                  │
│  │      namespace=user_id,                                   │
│  │      vector=embed(query),                                 │
│  │      top_k=5,                                             │
│  │      filter={"type": ["fact", "preference"]}              │
│  │    )                                                      │
│  ├─ 输出: context {                                          │
│  │     "chat_history": [...],                                │
│  │     "user_profile": {...},                                │
│  │     "retrieved_memories": [...]                           │
│  │   }                                                       │
│  └─ 超时: Redis/Pinecone 超时 >2s 则跳过，记录 warning 日志    │
└────────────────┬────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  节点 2: 意图识别与任务规划 (Planner)                        │
│  ├─ 输入: query + context                                    │
│  ├─ 模型: GPT-4o (temperature=0.1, response_format=json)    │
│  ├─ Prompt 模板 (见下方完整代码)                              │
│  ├─ 处理:                                                    │
│  │  • 调用 LLM 生成 JSON:                                    │
│  │    {                                                     │
│  │      "intent": "query/compare/analyze/chat",             │
│  │      "needs_retrieval": true/false,                      │
│  │      "sub_tasks": [                                      │
│  │        {"id": 1, "desc": "...", "tool": "xxx", "query": "..."}, │
│  │        ...                                               │
│  │      ],                                                  │
│  │      "dependencies": [[1,2], [3]]  # 任务依赖关系          │
│  │    }                                                     │
│  │  • 校验 JSON schema (jsonschema 库)                       │
│  │  • 如果 needs_retrieval=false → 跳转到节点 5              │
│  ├─ 输出: plan {intent, sub_tasks, dependencies}             │
│  └─ 降级: LLM 超时/失败 → 默认计划: [{"tool": "fallback_chat"}] │
└────────────────┬────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  节点 3: 工具路由与执行器 (Tool Executor)                    │
│  ├─ 输入: plan.sub_tasks                                     │
│  ├─ 工具注册表 (全局单例):                                    │
│  │  tools = {                                               │
│  │    "rag_search": RAGTool(config=rag_config),             │
│  │    "sql_query": SQLTool(config=db_config),               │
│  │    "web_search": WebSearchTool(api_key=xxx),             │
│  │    "calculator": CalculatorTool(),                       │
│  │    "code_interpreter": CodeTool(sandbox="docker"),       │
│  │    "fallback_chat": ChatTool(llm=llm),                   │
│  │  }                                                       │
│  ├─ 处理:                                                    │
│  │  【并行调度】                                              │
│  │  • 根据 dependencies 构建 DAG (networkx 库)                │
│  │  • 拓扑排序 → 可并行任务放入 asyncio.gather()             │
│  │                                                          │
│  │  【单个工具执行流程】                                      │
│  │  def execute_task(task):                                 │
│  │    tool = tools[task["tool"]]                            │
│  │    try:                                                  │
│  │      # 1. 参数校验                                        │
│  │      validated_input = tool.validate_input(task["query"])│
│  │      # 2. 执行 (带超时)                                   │
│  │      result = await asyncio.wait_for(                    │
│  │        tool.run(validated_input), timeout=30             │
│  │      )                                                   │
│  │      # 3. 结果缓存 (短期记忆)                              │
│  │      redis.setex(                                        │
│  │        f"task_cache:{session_id}:{task['id']}",          │
│  │        ttl=300, value=json.dumps(result)                 │
│  │      )                                                   │
│  │      return {"task_id": task["id"], "status": "ok", "data": result} │
│  │    except ToolError as e:                                │
│  │      return {"task_id": task["id"], "status": "error", "msg": str(e)} │
│  │    except TimeoutError:                                  │
│  │      return {"task_id": task["id"], "status": "timeout"} │
│  │                                                          │
│  │  【结果聚合】                                              │
│  │  • 收集所有任务结果 → results_list                       │
│  │  • 标记失败任务，供反思节点处理                           │
│  ├─ 输出: execution_results [ {task_id, status, data}, ... ] │
│  └─ 熔断: 同一工具连续失败 3 次 → 临时禁用 5 分钟 (Redis 计数器) │
└────────────────┬────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  节点 4: 反思与迭代控制器 (Reflector)                        │
│  ├─ 输入: plan + execution_results + context                 │
│  ├─ 模型: GPT-4o-mini (轻量模型，降低成本)                   │
│  ├─ Prompt 模板:                                             │
│  │  """                                                     │
│  │  你正在评估任务执行结果。请回答:                          │
│  │  1. 所有必要信息是否已收集？(yes/no)                      │
│  │  2. 如果有缺失，需要补充什么？(具体描述)                   │
│  │  3. 是否需要重新规划？(yes/no)                           │
│  │  4. 如果需要，生成新的 sub_tasks (JSON 格式)              │
│  │                                                          │
│  │  当前计划: {plan}                                        │
│  │  执行结果: {execution_results}                           │
│  │  可用工具: {tool_descriptions}                           │
│  │  """                                                     │
│  ├─ 处理:                                                    │
│  │  • 解析 LLM 输出 (JSON)                                   │
│  │  • 如果 "需要补充" → 生成新任务 → 回到节点 3 (最多 3 轮)    │
│  │  • 如果 "信息充足" → 进入节点 5                           │
│  │  • 如果 3 轮仍不足 → 标记部分失败，进入节点 5 生成降级回答  │
│  ├─ 输出: final_results {complete: bool, results: [...], missing: [...]} │
│  └─ 保护: 最大迭代次数=3，避免死循环                          │
└────────────────┬────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  节点 5: 答案生成器 (Answer Generator)                       │
│  ├─ 输入: query + final_results + context                    │
│  ├─ 模型: GPT-4o (temperature=0.3)                          │
│  ├─ Prompt 模板 (关键！):                                    │
│  │  """                                                     │
│  │  # 角色                                                  │
│  │  你是一个专业助手，基于提供的信息回答问题。                │
│  │                                                          │
│  │  # 约束                                                  │
│  │  1. 严格基于以下【检索结果】回答，不确定就说"无法确认"     │
│  │  2. 每个事实必须标注来源 [1][2]，格式: [来源 ID]          │
│  │  3. 如果检索结果冲突，说明差异并给出最可信的判断          │
│  │  4. 回答长度: {max_tokens} tokens 以内                    │
│  │  5. 语气: {user_profile.tone} (如: 专业/简洁/友好)        │
│  │                                                          │
│  │  # 用户问题                                              │
│  │  {query}                                                 │
│  │                                                          │
│  │  # 对话历史 (最近 3 轮)                                   │
│  │  {chat_history_summary}                                  │
│  │                                                          │
│  │  # 检索结果                                              │
│  │  {% for item in retrieved_results %}                     │
│  │  [{{item.id}}] {{item.source}}: {{item.content}}         │
│  │  {% endfor %}                                            │
│  │                                                          │
│  │  # 用户偏好                                              │
│  │  {user_profile}                                          │
│  │  """                                                     │
│  ├─ 处理:                                                    │
│  │  • 渲染 Prompt (Jinja2)                                  │
│  │  • 调用 LLM (streaming=True 支持流式输出)                 │
│  │  • 后处理:                                               │
│  │    - 验证引用格式 [1][2] 是否存在于 results              │
│  │    - 过滤敏感词 (同节点 0)                               │
│  │    - 添加元数据: {model, tokens_used, latency}           │
│  ├─ 输出: answer {                                          │
│  │     "content": "回答文本",                                │
│  │     "citations": [{"id":1, "source":"...", "url":"..."}],│
│  │     "meta": {"model":"gpt-4o", "tokens":1234}            │
│  │   }                                                      │
│  └─ 降级: LLM 失败 → 返回模板化回答 + 建议重试                 │
└────────────────┬────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  节点 6: 记忆写入器 (Memory Writer)                          │
│  ├─ 输入: request + answer + execution_results               │
│  ├─ 处理:                                                    │
│  │  【短期记忆 - 异步写入】                                    │
│  │  • 追加当前轮: Redis RPUSH session:{sid}:messages         │
│  │    [{"role":"user","content":query},                     │
│  │     {"role":"assistant","content":answer.content}]       │
│  │  • 设置 TTL: 24 小时无活动自动过期                         │
│  │                                                          │
│  │  【长期记忆 - 选择性写入】                                  │
│  │  • 触发条件 (满足任一):                                   │
│  │    1. 用户明确陈述事实: "我在北京工作"                    │
│  │    2. 用户表达偏好: "我喜欢简洁的回答"                    │
│  │    3. 任务产生新知识点 (通过 LLM 分类)                     │
│  │                                                          │
│  │  • 写入流程:                                              │
│  │    1. 用 LLM 提取关键信息 (小模型 + 专用 prompt)           │
│  │       Prompt: "从以下对话提取需要长期记忆的事实/偏好..."  │
│  │    2. 重要性评分 (1-10)，<5 则丢弃                        │
│  │    3. 向量化: embedding = embed(extracted_text)          │
│  │    4. 写入 Pinecone:                                     │
│  │       upsert(                                             │
│  │         id=uuid(),                                      │
│  │         vector=embedding,                                │
│  │         metadata={                                       │
│  │           "user_id": user_id,                            │
│  │           "content": extracted_text,                     │
│  │           "type": "fact/preference",                     │
│  │           "confidence": score/10,                        │
│  │           "timestamp": now()                             │
│  │         }                                                │
│  │       )                                                  │
│  │    5. 同时写入 PostgreSQL 结构化表 (便于管理/导出)         │
│  │                                                          │
│  │  【异步执行】                                              │
│  │  • 使用 Celery / asyncio.create_task() 后台执行           │
│  │  • 主请求不等待记忆写入完成                               │
│  ├─ 输出: (无，后台任务)                                     │
│  └─ 保护: 写入失败记录 error 日志，不影响主流程返回            │
└────────────────┬────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────┐
│  节点 7: 响应组装与返回 (Response Builder)                   │
│  ├─ 输入: answer + meta                                      │
│  ├─ 处理:                                                    │
│  │  • 组装标准响应格式:                                      │
│  │    {                                                     │
│  │      "code": 200,                                        │
│  │      "data": {                                           │
│  │        "answer": answer.content,                         │
│  │        "citations": answer.citations,                    │
│  │        "suggestions": ["相关问题 1", "相关问题 2"]       │
│  │      },                                                  │
│  │      "meta": {                                           │
│  │        "session_id": session_id,                         │
│  │        "tokens_used": answer.meta.tokens,                │
│  │        "latency_ms": total_latency,                      │
│  │        "model": answer.meta.model                        │
│  │      }                                                   │
│  │    }                                                     │
│  │  • 记录访问日志 (结构化: user_id, query, latency, status) │
│  │  • 埋点上报 (Prometheus / Datadog)                       │
│  ├─ 输出: JSON 响应                                          │
│  └─ 压缩: 响应 >1KB 且 Accept-Encoding 支持 → gzip 压缩      │
└─────────────────────────────────────────────────────────────┘
